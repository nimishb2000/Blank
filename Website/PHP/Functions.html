In-Built Functions



String Functions

Functions are reusable pieces of code that can be used throughout an applications, thus reducing amount of coding and making the applications modular.

PHP provides a rich collection of built-in functions. They can be called anywhere in our scripts wthout defining them.

PHP has a number of built-in String Manipulation Functions, supporting various kinds of string operations:

  • strtoupper() - This function accepts a string as an argument and returns it with all the alphabetic characters coverted to upper case.
  • strtolower() - This function accepts a string as an argument and returns it with all the alphabetic characters coverted to lower case.
  • strlen() - This function returns the length of the given string.
  • ltrim() - This function removes whitespace or other characters form the beginning of a string.
  • rtrim() - This function removes whitespace or other characters form the end of a string.
  • trim() - This function removes whitespace or other characters form both beginning and end of a string.
  • substr() - This function returns the specified portion from a sting.
  • strrev() - This function reverses a string.
  • strcmp() - This function compares two strings and returns -1 if first string is less, 0 if both strings are equal and 1 if first string is greater. This function is case-sensitive.
  • strcasecmp() - This function performs case-insensitive string comparison.
  • strncasecmp() - This function performs case-insensitive sring comparison of first n characters.
  • stripslashes() - This function removes backslashes from a string.

Date Function

The date() functions returns the current date/time in different formats depending on the parameters specified. Its is defined as date(format).

The format parameter specifies how the to format the date/time. Characters used are:

  • d - Returns day of the month (01 to 31)
  • D - Short textual representation of day of week (Mon to Sun)
  • l - Full textual representation of day of week (Monday to Sunday)
  • m - Represents month number (01 to 12)
  • M - Short textual representation of month (Jan to Dec)
  • F - Full textual representation of month (January to Decemeber)
  • y - Two-digit representation of year
  • Y - Four-digit representation of year
  • h - 12-hour format of an hour
  • H - 24-hour format of an hour
  • i - Minutes
  • s - Seconds
  • a - Lowercase Ante Meridiem or Post Meridiem (am or pm)
  • A - Uppercase Ante Meridiem or Post Meridiem (AM or PM)

Getdate Function

This funtions returns date/time information in the form of an associative array.

If a timestamp has been given as a parameter, its value is returned else the current local time is returned.

Elements returned in the associative array are as follows:

Key Description
seconds Seconds past the minutes (0-59)
minutes Minutes past the hour (0-59)
hours Hours of the day (0-23)
mday Day of the month (0-31)
wday Day of the week (0-6)
mon Month of the year (1-12)
year Year (4 digits)
yday Day of the year (0-365)
weekday Day of the week
month Month of the year
0 Unix Timestamp

Gettimeofday Function

This function returns the current time in the form of an associative array.

Elements in the returned associative array are:

  • sec - Seconds since Unix Epoch
  • usec - Microseconds
  • minuteswest - Minutes west of Greenwich
  • dsttime - Type of dst correction

Defining Functions

A function is a group of code statements that can be called by an associated name to accomplish a specific task or purpose in a program.

Functions are declared using the function keyword, followed the the name of the function and the parameters (if any) in parentheses seperated by commas.

Passing Arguments

Arguments in a function can be passed in two ways:

  • By value
  • By reference

While passing arguments by value, a copy of the original argument is created and passed to the called function. Hence, any changes made inside the function will not affect the original value of the argument. This is the default method.

While passing arguments by reference, the actual memory address of the argument is passed to the called function. Any changes made inside the called function will modify the original argument. An ampersand (&) is prefixed before the argument name.
